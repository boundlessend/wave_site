# PRD - Длина волны (Wavelength online)

Статус: живой документ. Раздел «Что есть» описывает текущее состояние прод-версии,
раздел «Планы» - незакреплённый backlog идей (приоритеты не расставлены).

Прод: https://wavesite-rho.vercel.app
Репозиторий: https://github.com/boundlessend/wave_site

## 1. Цель

Онлайн-версия настольной «Длины волны» (Wavelength): открыл ссылку - играешь компанией,
у каждого своё устройство. Без регистрации, без своего сервера, бесплатный хостинг.

## 2. Что уже есть (реализовано)

### Геймплей
- Два режима: соревнование (две команды) и кооператив (все против колоды из 7 карт).
- Полные правила: фаза телепата, фаза команды (стрелка), фаза «слева/справа» для соперников,
  раскрытие, начисление очков (2-4 за зону, +1 за верную сторону), правило догоняющего.
- Победа по 10+ очкам; в коопе центр даёт 3 очка.
- Встроенный русский набор карт-противоположностей.

### Мультиплеер и синхронизация
- Комнаты по ссылке-приглашению, realtime через Supabase (broadcast + presence).
- Модель host-authoritative без своего сервера; авторитет (host) выбирается по presence
  и мигрирует при отключении.
- Вышедшие игроки удаляются автоматически, зависший раунд можно пропустить, выход в лобби.

### Безопасность (по итогам пентеста)
- Секрет мишени честный: позиция зоны живёт только на устройстве телепата до раскрытия.
- Realtime-сообщения подписаны HMAC-SHA256 секретом комнаты; подпись покрывает тип события,
  отправителя и время (+ окно свежести и отсев повторов против replay). Секрет в hash ссылки
  (`#k=`), не уходит на сервер. Чужой без ссылки не подделает состояние.
- Presence подписывается целиком; роль актёра привязана к устройству-отправителю; входящие
  сообщения ограничены по частоте (анти-флуд). Код комнаты 6 символов на CSPRNG.
- Security-заголовки (CSP запинен на поддомен проекта и пр.) в `vercel.json`. Регистрация в Supabase отключена.
- Полный аудит проекта + пентест: см. `AUDIT.md` (27 пунктов, все закрыты 2026-07-01).

### Дизайн и UX
- Редакторский («эдиториал») вид: светлая бумага, строгая типографика (Unbounded / Fraunces /
  Inter / JetBrains Mono), красно-синий акцент, жёсткая сетка.
- Переключаемый фоновый 3D (чистый three.js, ленивый чанк): 5 сцен на тему волны
  (ридж-волны по умолчанию, полутон, фигуры, каркас, рябь) + «без фона». Выбор в localStorage.
- Анимации (Motion + GSAP), звук, конфетти на победе.
- Доступность: шкала как slider (стрелки/Home/End), не-цветовые метки команд, `prefers-reduced-motion`.
- Адаптив: мобильная и десктоп-раскладки.

### Инфраструктура и качество
- React 19 + Vite + TypeScript (строгий). Логика игры - чистые функции, self-check'и правил и движка.
- CI (GitHub Actions): tsc + self-check + lint + build на каждый push/PR.
- Авто-деплой на Vercel при push в `main`. BSD 3-Clause, README актуален.

### Тарифы и ограничения (текущие)
- Vercel Hobby: статика на CDN, лимит по трафику (~100 ГБ/мес), не по размеру бандла.
- Supabase Free: лимиты на realtime-сообщения/одновременные соединения; фон и анимации
  клиентские, на Supabase не влияют. Realtime через `@supabase/realtime-js` (без лишних частей SDK).
- Бандл: стартовая загрузка ~133 КБ gzip (вендоры react/supabase/motion — отдельными чанками,
  правки кода их не инвалидируют); 3D-сцена отдельным ленивым чанком ~133 КБ gzip.

## 3. Планы / backlog (черновые идеи, разгрести позже)

### 3.1 Оптимизация
- Сделано в аудите (см. `AUDIT.md`): vendor-чанки, `@supabase/realtime-js` вместо supabase-js,
  троттлинг 3D-фона до ~30 fps, обрезка весов шрифтов, `strict` в TS.
- Полный аудит Lighthouse / Core Web Vitals; async-загрузка шрифтов (сейчас блокируется CSP
  `script-src`), `size-adjust` для fallback-метрик (CLS).
- 3D ещё легче: инстансинг для точечных сцен, не стартовать сцену до первого кадра контента.
- Свой favicon и og-превью под эдиториал; мета-теги для шеринга.

### 3.2 Новые фичи (геймплей)
- Больше карт и категорий; пользовательские наборы (добавить/импортировать свои пары).
- История раундов и итоговая таблица партии.
- Опциональный таймер на подсказку/ход.
- Кастомные названия команд, простые аватары/эмодзи игроков.
- Лёгкие реакции (эмодзи) во время раунда.

### 3.3 UX и качество
- Короткий онбординг/туториал для новичков; подсказки прямо в фазах.
- Шеринг результата партии, QR-код комнаты для быстрого входа с телефона.
- Локализация (i18n), для начала RU/EN.
- Тёмная тема как опция (переключатель уже есть - расширить на тему, не только фон).

### 3.4 Техническое и надёжность
- E2E-тесты (Playwright) на ключевые сценарии: создание комнаты, вход по ссылке, полный раунд.
- Устойчивое восстановление состояния при реконнекте/сворачивании вкладки.
- Лимит размера комнаты (рейт-лимит входящих уже сделан — см. `AUDIT.md`).
- Телеметрия по желанию (анонимная, без личных данных) для понимания узких мест.
- Per-device ключи / серверная авторизация ролей (сейчас приглашённые доверены by design).
